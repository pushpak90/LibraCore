LibraCore

Secure Library Inventory & Lending Management API

Problem Statement

LibraCore is a backend RESTful API designed to manage the digital operations of a modern library system, including book cataloging, user access control, and lending workflows.

The system replaces manual or semi-automated library processes with a secure, scalable, and role-based backend service that ensures accurate tracking of books, controlled borrowing, and proper enforcement of library policies.

The application is built using Spring Boot, persists data using MySQL, and implements JWT-based stateless authentication to support secure access and horizontal scalability.

Business Objective

  • Maintain a centralized book catalog
  • Control book borrowing and return workflows
  • Enforce role-based permissions
  • Prevent over-issuing of books
  • Demonstrate real-world backend best practices

Assumptions & Constraints

  • Each book can have multiple physical copies
  • Users can borrow one copy per request
  • Borrowing duration and fines are handled externally
  • Librarians manage books globally
  • The system focuses only on inventory and lending

User Roles

Role Description
MEMBER Browse books and borrow available copies
LIBRARIAN Manage book inventory and process returns
ADMIN Onboard librarians and remove books

Functional Requirements

Authentication & Authorization

  • User registration using email and password
  • Passwords hashed using BCrypt
  • Default role assigned as MEMBER
  • JWT token generated on login/registration

Book Catalog Management

  • ISBN, Title, Author, Category
  • Total copies and available copies tracking
  • Public access to book listings
  • Admin-only book creation and deletion
  • Librarian-only book updates

Lending Management

  • Members can borrow one book per request
  • Borrowing allowed only if copies are available
  • Librarians process book returns
  • Availability updates automatically

REST API Endpoints

Authentication

POST /api/auth/register
POST /api/auth/login

Books

GET /api/books
POST /api/books (ADMIN)
PUT /api/books/{id} (LIBRARIAN)
DELETE /api/books/{id} (ADMIN)

Lending

POST /api/books/{id}/borrow (MEMBER)
PUT /api/loans/{id}/return (LIBRARIAN)
